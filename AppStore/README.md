# App Store Release Pack

Materials for submitting **Jokester Deck** to the App Store from a personal developer account.

## Files

| File | Purpose |
|------|---------|
| [metadata-en.md](metadata-en.md) | Description, promotional text, subtitle, keywords, icon prompt, reviewer notes, screenshot plan |
| [privacy-policy.md](privacy-policy.md) | Privacy policy source (markdown) |
| [../docs/privacy-policy.html](../docs/privacy-policy.html) | Host-ready privacy policy page |

## App icon

1024×1024 icon added to [`jokester-deck/Assets.xcassets/AppIcon.appiconset/`](../jokester-deck/Assets.xcassets/AppIcon.appiconset/).

## Before submission checklist

1. **Support email** — Confirm `support@jokester.deck` is active (used in reviewer notes and privacy policy). Update all three files if you use a different address.
2. **Privacy Policy URL** — Publish `docs/privacy-policy.html` (e.g. GitHub Pages) and paste the HTTPS URL into App Store Connect.
3. **App Store Connect** — Copy fields from `metadata-en.md` into the listing.
4. **Reviewer notes** — Paste the "Notes for App Review" section into App Review Information (includes screenshot / marketing framing note).
5. **Screenshots** — Follow section 8 in `metadata-en.md`: use Codex + Collection frames; replace slot-reel art with Demo + Trivia; crop iPad Home.
6. **Categories** — Primary: Education · Secondary: Reference · Age: 4+

## Suggested GitHub Pages privacy URL

If this repo is published as `https://github.com/USER/jokester-deck` with Pages from `/docs`:

```
https://USER.github.io/jokester-deck/privacy-policy.html
```

Replace `USER` with your GitHub username before submitting.
